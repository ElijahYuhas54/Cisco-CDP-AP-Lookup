# ap_lookup.py

Connects to one or more Cisco switches via SSH, runs `show cdp neighbors`, and identifies any access points in the neighbor table.

## Credentials File

The script reads credentials from `scripts/credentials/switch-credentials.txt`. No credentials are stored in the code.

### Format

```
# Primary SSH credentials (required)
Username: <username>
Password: <password>

# Enable mode secret (required)
Secret: <enable_secret>

# Secondary SSH credentials ‚Äî fallback if primary auth fails (optional)
SecondaryUsername: <secondary_username>
SecondaryPassword: <secondary_password>
```

| Field               | Required | Description                                        |
|---------------------|----------|----------------------------------------------------|
| `Username`          | Yes      | Primary SSH login username                         |
| `Password`          | Yes      | Primary SSH login password                         |
| `Secret`            | Yes      | Enable mode secret                                 |
| `SecondaryUsername` | No       | Fallback username if primary auth fails            |
| `SecondaryPassword` | No       | Fallback password if primary auth fails            |

Lines beginning with `#` are treated as comments and ignored.

## Switch Database File

`scripts/references/switches.txt` maps buildings and campuses to their switch IPs. The script reads this file at runtime to resolve building numbers and campus names to IP lists.

### Format

```
[Building 10]
192.168.10.1
192.168.10.2

[SiteName]
10.0.1.1
10.0.1.2
```

Buildings are labeled `[Building <number>]`. Remote sites or campuses can use any label (e.g. `[SiteName]`).

## Usage

Run the script, then enter your targets one per line. Press **Enter on a blank line** to finish and begin scanning.

```
Enter switch IPs, building numbers, ranges, or site names
(one per line, blank line when done):
  Examples: 192.168.10.5  |  10  |  building 10  |  5-15  |  SiteName
>
```

### Accepted Input Formats

All input is **case-insensitive**.

| Input | Resolves to |
|-------|-------------|
| `192.168.10.5` | That single IP |
| `10` | All switches in Building 10 |
| `building 10` | All switches in Building 10 |
| `Building 10` | All switches in Building 10 |
| `5-15` | All switches in Buildings 5 through 15 |
| `building 5 - building 15` | All switches in Buildings 5 through 15 |
| `SiteName` | All switches under that site label |

**Comma-separated lists on a single line are supported:**
```
> 12, building 7, 3, 1, 20, 25
```

Ranges are reversible ‚Äî `15-5` is treated the same as `5-15`.

Duplicate IPs across multiple entries are automatically removed before scanning.

## AP Detection

The script matches two AP naming formats in CDP output:

### 1. Site Naming Convention

Format: `<site><building_number><building_letters><room>ap<jack>`

| Segment          | Example  | Description                               |
|------------------|----------|-------------------------------------------|
| Site code        | `abc`    | 2‚Äì4 lowercase letters                     |
| Building number  | `10`     | 1‚Äì3 digits                                |
| Building letters | `cd`     | 1‚Äì4 lowercase letters                     |
| Room             | `101`    | 1‚Äì4 digits                                |
| Literal `ap`     | `ap`     | Always present                            |
| Jack             | `1234a`  | 1‚Äì6 digits + optional letter, or `n/n/n`  |

**Examples:**
- `abc10cd101ap1234`
- `abc10cd101ap1234a`
- `abc10cd101ap1/0/1`

### 2. MAC-Based Default Name

Format: `AP` followed by three dotted groups of 4 hex characters.

**Example:** `AP54fa.3b9c.9487`

This format appears when an AP has not yet been renamed from its factory default in the WLC or controller.

## Authentication Flow

1. Tries **primary** credentials (`Username` / `Password`)
2. Falls back to **secondary** credentials (`SecondaryUsername` / `SecondaryPassword`) if primary auth fails
3. If both fail, the switch is skipped with an error message and scanning continues

## Setting Up the `ap_lookup` Shorthand (Windows)

To run the script by typing `ap_lookup` in any terminal:

1. Create a folder for global scripts (e.g. `C:\tools`) and add it to your system `PATH`:
   - Search **"Edit the system environment variables"** ‚Üí **Environment Variables**
   - Under **System variables**, select `Path` ‚Üí **Edit** ‚Üí **New** ‚Üí add your folder path

2. Create `ap_lookup.bat` in that folder:
   ```bat
   @echo off
   python "C:\path\to\scripts\python\ap_lookup.py" %*
   ```

3. Open a new terminal and run `ap_lookup`.

**PowerShell alternative** ‚Äî add to your profile (`notepad $PROFILE`):
```powershell
function ap_lookup { python "C:\path\to\scripts\python\ap_lookup.py" $args }
```
The `.bat` approach works in both CMD and PowerShell; the function only works in PowerShell.

## Troubleshooting

| Problem | Likely Cause | Fix |
|---------|--------------|-----|
| `[ERROR] Credentials file not found` | File missing or wrong path | Verify file exists at `scripts/credentials/switch-credentials.txt` |
| `[ERROR] Missing required credential field` | `Username`, `Password`, or `Secret` line missing | Add the missing field to the credentials file |
| `[ERROR] Switch database not found` | `switches.txt` missing | Verify file exists at `scripts/references/switches.txt` |
| `[!] No switches found for 'Building X'` | Building not in database | Add the building section to `switches.txt` |
| `[!] Connection failed: Authentication` | Wrong credentials | Check credentials file; verify secondary credentials are set if needed |
| `[!] Connection failed: Timed out` | Switch unreachable or SSH not enabled | Check IP reachability; verify `ip ssh version 2` is configured |
| No APs found but you expect some | CDP name truncated or CDP disabled | Run `show cdp neighbors detail` manually to check full hostnames; verify CDP is enabled (`cdp run` / `cdp enable`) |

---

## Notes

- `show cdp neighbors` (short form) may truncate long device names. If AP names are being cut off, swap the command in `process_switch()` for `show cdp neighbors detail`.
- MAC-based AP names (`AP54fa.3b9c.9487`) indicate the AP has not yet been renamed in the controller.
- CDP must be enabled globally (`cdp run`) and per-interface (`cdp enable`) for neighbors to appear.
- Switches are scanned sequentially. Runtime scales linearly with switch count and response time.
